# EVAL_QUESTIONS.md — CareerFit AI 자기 평가 질문
## 이 질문들에 스스로 답할 수 있어야 최종 발표와 면접에 나설 수 있다.

## 프로젝트 전체
- 이 서비스가 해결하는 문제는 무엇인가?
  - 공고를 찾기 위해 여러 사이트를 돌아다녀야 하는 문제
  - 공고 내용을 파악하기 위해 여러 사이트를 돌아다녀야 하는 문제
  - 공고를 보고 나에게 맞는 공고인지 파악하기 위해 
- RAG를 사용하지 않으면 어떤 문제가 생기는가?
  - 일반 LLM을 사용하면 사실을 기반으로 정확한 답변을 하지 못할 수 있다.
- 가장 어려웠던 구현은 무엇이고, 어떻게 해결했는가?
  - LangChain의 LangGraph를 사용하여 RAG 파이프라인을 구축하는 과정이 어려웠다.
  - RAG 파이프라인을 구축한 후 RAG의 성능을 개선하는 과정이 어려웠다.


## 데이터 관련
- 왜 이 데이터를 선택했는가?
- 데이터의 어떤 부분이 신뢰할 수 없을 수 있는가?
  - 데이터의 업데이트 시점에 따라 정확성이 떨어질 수 있음
  - 데이터 수집 과정에서의 편향이나 오류가 있을 수 있음
- SQLite와 ChromaDB에 같은 데이터를 저장하는 이유는 무엇인가?
  - SQLite는 RDBMS로 SQL을 사용할 수 있기 때문
  - ChromaDB는 벡터 데이터베이스로 검색을 위해 사용
- 결측치와 중복을 어떻게 처리했는가?


## RAG 관련
- RAG 구조에서 ChromaDB는 어떤 역할을 하는가?
  - LLM이 검색 결과를 기반으로 답변을 생성할 수 있도록 도와줌
- n_results 값을 높이면 어떤 트레이드오프가 생기는가?
  - 검색 결과의 양이 늘어나지만, 관련성이 떨어지는 결과가 포함될 수 있음
  - 답변의 정확성이 향상될 수도 있지만, 비용이 증가하고 속도가 느려질 수 있음
- 검색 결과 품질을 어떻게 검증했는가?
  - 검색 결과를 LLM에 전달하여 답변을 생성했을 때, 답변의 정확성과 관련성을 확인했음
- Hallucination이 무엇이며, RAG가 이를 어떻게 줄이는가?
  - LLM이 사실이 아닌 내용을 사실처럼 지어내는 현상
  - RAG는 검색 결과를 LLM 기반으로 답변을 생성하기 때문에 hallucination을 줄일 수 있음

  - POST는 body에 데이터를 전달할 수 있기 때문
- 응답 JSON에 sources를 포함시킨 이유는 무엇인가?
  - 사용자가 답변의 근거를 확인할 수 있도록 하기 위해
- /health 엔드포인트가 왜 필요한가?
  - API 서버가 정상적으로 동작하는지 확인할 수 있기 때문


## 보안 관련
- .env 파일을 GitHub에 올리면 어떤 일이 생기는가?
  - API 키 등이 노출될 수 있음

  - .env: API 키 등 민감한 정보 포함

## Docker 관련
- Docker를 사용하는 이유는 무엇인가?
  - 개발 환경을 일관되게 유지하기 위해

  - CMD: 컨테이너가 시작될 때 실행할 명령어
